# ADR-XXXX: Short imperative title

- **Status**: Proposed | Accepted | Rejected | Superseded by ADR-YYYY
- **Date**: YYYY-MM-DD
- **Author(s)**: Full name (organization)
- **Deciders**: Founder and any additional CODEOWNERS
- **Related**: Links to prior ADRs, issues, PRs, or external documents

## Context

The situation, technical facts, and constraints that make this decision necessary.
State the problem, not the solution.

## Decision

The decision, stated as a clear, testable normative sentence.

## Consequences

- Positive: what improves.
- Negative: what regresses, what becomes more complex.
- Neutral: what changes without net impact.

## Locked implications

Anything that becomes irrevocably tied to this decision — brand terms, file
formats, wire protocols, API signatures. Changing a locked implication
requires a new ADR that explicitly supersedes this one.

## Alternatives considered

Each alternative with a short rationale for rejection.

## References

External links, papers, prior art.
