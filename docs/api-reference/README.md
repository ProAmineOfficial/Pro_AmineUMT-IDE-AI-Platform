# UMT VR API Reference

The UMT VR API exposes 45 standardized interfaces spanning serial
communication, USB, audio, analog, camera and display, PWM and timing,
networking, debug, and system.

- `vr-interfaces.md` — the complete interface catalog.
- `three-levels.md` — the three-level API (Interface, Interface_Pin,
  Digital_Pin).
- `naming-rule.md` — the naming rule (short interface name for
  `Interface()`, direction-suffixed for `Interface_Pin` and `Digital_Pin`).
- `lifecycle.md` — the four lifecycle methods (activate, deactivate,
  enable, disable) with state transitions.

For per-target physical pin mappings, see `boards/*/README.md`.
