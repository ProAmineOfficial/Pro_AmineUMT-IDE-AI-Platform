# Roadmap

The Pro_AmineUMT IDE with AI Platform roadmap is organized into three
horizons: near-term (0–12 months), mid-term (12–24 months), and long-term
(24+ months). Dates are aspirational; final release depends on community
adoption and partner validation.

## Horizon 1 — Foundation (0–12 months)

- Complete repository bootstrap and public open-source launch.
- Publish the UMT BGA 16×16 Package Substrate Standard v1.0.
- Release the AIA Engine reference implementation for three target boards:
  NanoKit Integrated ESP32, Arduino UNO R3, and Raspberry Pi Pico.
- Ship the Pro_AmineUMT IDE (VS Code extension) with editor, debugger,
  monitor, and AI Bashir sidecar.
- Publish the five-article launch series: problem statement, UMT
  introduction, AIA Engine, BGA Standard, IDE roadmap.
- Onboard the first ten community contributors.

## Horizon 2 — Adoption (12–24 months)

- Add support for the STM32 Nucleo family and the Raspberry Pi RP2350.
- Add the NanoKit-iM (MCU or SoC Integrated Modules) production board.
- Ship the visual block editor (Node-RED style) for bidirectional
  code-and-diagram editing.
- Publish the AIA Engine as a language-neutral protocol so that alternative
  front-ends (Kotlin, Rust) can plug in.
- First commercial engagement — one Tier 2 or Tier 3 partner in the
  Enterprise directory.

## Horizon 3 — Standardization (24+ months)

- Present the UMT BGA 16×16 Package Substrate Standard for review by an
  industry standards body (JEDEC, IPC, or equivalent).
- Announce first chipmaker adoption of the UMT VR pinout at the package
  substrate stage.
- Deliver the NVIDIA UMT-code proof-of-concept demonstrated at MVP.
- Release Pro_AmineUMT IDE with AI v1.0.
